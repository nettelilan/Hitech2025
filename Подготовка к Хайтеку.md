## Изучение законодательства

### 187-ФЗ
Что требуется предусмотреть в соответствии со 187-ФЗ "О безопасности критической информационной инфраструктуры":

1. Для разработки ОРД можно взять:
	* определения АСУ, КИИ, компьютерная атака, компьютерный инцидент;
	* основные задачи системы безопасности значимого объекта КИИ
2. Необходимо категорировать. Существует 1,2,3 категории значимости. Если объект КИИ не соответствует критериям значимости, показателям этих критериев и их значения, ему не присваивается ни одна из этих категорий.
3. Сведения о категорировании необходимо в 10-дневный срок отправить во ФСТЭК.
4. Субъекты КИИ должны незамедлительно информировать о компьютерных атаках и компьютерных инцидентах.
5. Использовать на значимых объектах КИИ ПО, сведения о котором включены в единый реестр российских программ, которое используется в ГИС или иных гос.систем.
### 127 Постановление
Что требуется предусмотреть в соответствии с постановлением Правительства РФ №127 "Об утверждении Правил категорирования объектов КИИ РФ, а также перечня показателей критериев значимости объектов КИИ РФ и их значений":

1. Объекту КИИ по результатам категорирования присваивается категория значимости с наивысшим значением.
2. Для каждого показателя критериев значимости, для которого установлено более одного значение такого показателя (территория, количество людей), оценка производится по каждому из значений показателя критериев значимости, а категория присваивается по наивысшему значению такого показателя.
3. В случае если ни один из показателей неприменим для объекта КИИ не соответствует ни одному показателю, категория значимости не присваивается.
4. Устанавливаются 3 категории значимости. Самая высокая - первая, самая низкая - третья.
5. Должна быть создана постоянно действующая комиссия по категорированию.
6. Решение о категорировании оформляется актом, который в 10-дневный срок отправляется во ФСТЭК

## Перечень ОРД 

### КИИ

1. Решение (приказ) о создании постоянно-действующей комиссии по категорированию (Постановление 127)
2. Акт о категорировании (Постановление 127)
		Акт должен содержать сведения об объекте КИИ, сведения о присвоенной объекту КИИ категории значимости либо об отсутствии необходимости ему одной из таких категорий, и другие сведения в соответствии с п.17.
		
3. Политика/положение по обеспечению безопасности ОКИИ
4. Политика управления доступом, положение об управлении учетными записями и доступом пользователей (Приказ 239). 
		Должна быть описана модель доступа (матрица доступа), роли и права пользователей, требования к паролям (в приложении должен быть перечень разрешенных устройств)
5. Процедура предоставления удаленного доступа к управлению ЗОКИИ
6.  Политика/положение защиты от вредоносного ПО
		В приложении должен быть перечень запрещенного ПО
7. Положение по использованию съемных носителей информации
8. Положение о проведении контроля информационной безопасности
9. Процедура проведения мероприятий по оперативному реагированию и ликвидации последствий компьютерных атак и компьютерных инцидентов
10. Регламент обеспечения целостности и доступности в ЗОКИИ
11. Положение по использованию ПО 
12. Порядок резервного копирования и восстановления информации
13. Регламент защиты технических средств, систем и машинных носителей информации ЗОКИИ
14. Регламент взаимодействия подразделений ФСБ РФ и общества при осуществлении информационного обмена в области обнаружения, предупреждения и ликвидации последствий компьютерных атак
15. Положение о применении безопасных конфигураций ПО
16. Регламент планирования мероприятий по обеспечению безопасности ОКИИ
17. Регламент обеспечения действий в нештатных ситуациях на ЗОКИИ
18. Положение о повышении осведомленности работников по вопросам ИБ

## План мероприятий

1. Сведения (акт) о категорировании необходимо в 10-дневный срок отправить во ФСТЭК (187-ФЗ, постановление 127).
2. Переход на импортозамещенное ПО, если требуется. Разработка стратегии по импортозамещению (187-ФЗ).
3. Обучение сотрудников по вопросам ИБ (например, курсы по ИБ, статьи по ИБ, тестовые фишинговые рассылки)
4. На сервер KSC можно подключаться только с JH, на KSC необходимо настроить двухфакторную аутентификацию (в рамках практической части не настраивать, чтобы было легко проверить сделанные настройки)
